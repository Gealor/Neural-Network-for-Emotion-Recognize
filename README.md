# Цель проекта
Написать собственную нейронную сеть для задач распознавания звуков.
В данной ветке я продолжаю работать со сверточной нейронной сетью(CNN).

# Описание
В проекте представлено три python скрипта:
- download_dataset: для скачивания датасета RAVDRESS(https://www.kaggle.com/datasets/uwrfkaggler/ravdess-emotional-speech-audio?resource=download)
- prepare_data: для подготовки данных для обучения модели(с аугментацией)
- model_training: хранится непосредственная архитектура модели, а так же визуализация результатов обучения

# Предложения из предыдущей итерации
1. Использовать StandardScaler из sklearn, для нормализации данных
2. Использовать ReduceLROnPlateau (снижение LR, если валидационная потеря не уменьшается)
3. Использовать LSTM/GRU для учета временной последовательности

# Изменения
1. Добавлена нормализация данных(StandardScaler) и предварительный прогон по ним, т.к. mfcc-коэффициенты имеют разную высоту для разных аудиозаписей.
2. Переосмыслен подход к подготовке данных, теперь разделение на наборы определяется по дикторам, т.е. в два разных набора не может попасть один и тот же диктор.
3. Переход от mfcc-коэффициентов к мел-кепстральным, т.к они содержат более подробную информацию о текстуре и гармонике голоса.
4. Добавлен ReduceLROnPlateau, чтобы обучение замедлялось, если валидационные потери не улучшаются на протяжении определенного количества эпох
5. В классификаторе вместо Flatten слоя используется GlobalAveragePooling2D, что значительно снижает количество параметров
6. Переделаны сверточные блоки + добавлен четвертый блок для выявления более сложных закономерностях.

# Результаты
Модель прогонялась при следующих параметрах:
1. Количество эпох=200

С результатами обучения со всеми описанными изменениями можно ознакомиться в папке 200_epochs_trying_resolve_cnn_overlifting.

По результатам обучения можно сказать, что модель стала честно классифицировать именно эмоции, а не делать вывод на основе темба и голоса диктора, но при этом снизилась точность классификации. Также удалось если не избавиться, то значительно снизить переобучение модели(тестовые и валидационные графики идут рядом друг с другом). Сейчас основная задача стоит в том, чтобы повысить точность модели и снизить процент ошибок.

# Предполагаемые причины: 
1. Из текущей архитектуры выжат максимум и требуется переход к новой архитектуре CRNN (CNN с LSTM слоем), т.к. эмоция - это процесс, разворачивающийся во времени и простой картинки уже недостаточно.
2. Неоптимальные параметры модели(начальная скорость обучения, l2-легурялизация и т.д)
